Receipt Field Guide
claim_idUnique fingerprint for this check. Your retrievable proof of due diligence.
verified_atExact UTC timestamp the live government source was queried — not cached.
source_urlDirect link to the government database. Examiners can verify the source independently.
resultMATCH or NO_MATCH against this source at this exact moment in time.
match_score100% = exact. 85–99% = fuzzy — catches misspellings and name variations.
data_versionSHA-256 of the list version active at check time. Proves exactly what data was used.
How It Works
Step 1Your system queries Inner-Circle with an entity name via API.
Step 2We screen against all four live government sources simultaneously.
Step 3We return a tamper-evident receipt for each source — match or no match.
Step 4The receipt is stored with a cryptographic hash of the data version used.
Step 5Years later, examiners can retrieve that receipt and verify the exact check.
OFAC requires 5-year recordkeeping under 31 CFR Part 501. Inner-Circle generates receipts that satisfy that requirement — tied to the exact government data version active at the time of the check. Not a log entry. A tamper-evident record.
What Retrieval Proves
claim_idPermanent identifier. The same ID issued at check time, unchanged.
verified_atThe exact moment the original check ran. Immutable.
data_versionThe government list version active when the check ran. Not today's list.
resultMatch or no match at that exact moment. The record does not change if the list changes later.
When an examiner arrives, you retrieve — you don't reconstruct. The receipt was sealed at the time of the check. Retrieval returns exactly what was issued. No interpretation required.
Replay Field Guide
hash_verifiedConfirms the stored snapshot matches the hash recorded at ingestion. The data has not been modified.
result_consistentThe replay produced the same result as the original check. The receipt is authentic.
computed_hashSHA-256 of the raw snapshot re-computed at replay time.
original_hashSHA-256 recorded at the time of the original check. Must match computed_hash.
event_hashCryptographic chain link to the immutable audit ledger.
Why Replay Matters
Log entrySays a check was run. Cannot be verified.
ReceiptProves what the result was. Tied to a specific data version.
ReplayProves the receipt is authentic. Re-runs the check against the original raw data snapshot.
Replay is the moat. Any vendor can produce a log entry. Inner-Circle replays the check against the exact government data that was live when it ran — and proves the result hasn't changed. No log entry does that.
